Add `## 🌍 Real-World Scenarios` to the requested topic when the user asks for practical scenarios.

For each scenario, keep this structure:

1. Context
2. User Flow
3. Technical Requirements
4. Edge Cases
5. Recommended Approach
6. Senior/Staff Tradeoffs
7. Testing Notes
8. Interview Answer
9. Giải thích tiếng Việt ngắn gọn

Rules:

- Scenarios must stay grounded in the topic.
- Keep them practical and short.
- Avoid repeating the same business case with different names.
